# TodoList示例项目介绍

在接下来的章节中，我们将使用React完成一个名为TodoList（待办）的示例项目。这个示例包括一个输入框，一个提交按钮以及一个无序列表。示例完成以后，我们可以在输入框中输入数据（待办项），点击提交按钮将数据添加到位于输入框下方的无序列表（待办列表）中。

# TodoList实现基本骨架

本节，我们将开始编写TodoList。

首先，我们要在src目录下创建一个TodoList.js文件，在这个文件里我们定义一个名为TodoList的组件。

import React,{ Component } from 'react';

class TodoList extends Component {
    render() {
        return (<div>todolist</div>)
    }
}

阅读代码可知，这个组件只是简单地返回了文本todolist。为了能在外部引用它，我们需要使用export default TodoList;命令将其导出。

import React,{ Component } from 'react';

class TodoList extends Component {
    render() {
        return (<div>todolist</div>)
    }
}
export default TodoList;

下面，我们在入口文件index.js中引用TodoList组件，将TodoList组件挂载到id为root的DOM节点下。

import React from 'react';
import ReactDOM from 'react-dom';
import TodoList from './TodoList';

ReactDOM.render(<TodoList />, document.getElementById('root'));

查看浏览器页面，可以看到页面显示文本todolist。

接下来，我们继续修改TodoList组件代码，添加一个输入框和提交按钮，并在输入框和提交按钮下添加一个列表。

import React,{ Component } from 'react';

class TodoList extends Component {
    render() {
        return (
        <div><input /><button>提交</button></div>
        <ul>
            <li>学英语</li>
            <li>Learning React</li>
        </ul>
        )
    }
}
export default TodoList;

此时，页面没有运行成功，抛出了一个错误。

这是因为上述写法违反了JSX语法，因为所有内容都必须包含在一个大的标签中。修改代码，在render方法返回的内容最外层再加上一个div标签，将全部内容包裹起来。

import React,{ Component } from 'react';

class TodoList extends Component {
    render() {
        return (
        <div>
            <div><input /><button>提交</button></div>
            <ul>
                <li>学英语</li>
                <li>Learning React</li>
            </ul>
        </div>
        )
    }
}
export default TodoList;

此时，再次查看页面，一切显示正常。

使用Chrome浏览器开发者工具查看，可以看到在root节点下存在一个div标签包含了页面的所有内容。

若我们不想在页面内容外层加上一个div标签，同时又满足JSX语法，那么我们可以从react中引入名为Fragment的占位符代替div标签。

import React,{ Component,Fragment} from 'react';

class TodoList extends Component {
    render() {
        return (
        <Fragment>
            <div><input /><button>提交</button></div>
            <ul>
                <li>学英语</li>
                <li>Learning React</li>
            </ul>
        </Fragment>
        )
    }
}
export default TodoList;

再次查看，外层的div标签已经消失了。

# TodoList实现输入待办功能

前面，我们已经完成了TodoList功能的基本结构，这一节，我们将实现TodoList基本功能：即在输入框输入待办事项数据，单击提交按钮后，数据自动添加到输入框下面的列表中。

在以前，我们通常是对提交按钮绑定一个事件，然后获取输入框里的数据，并找到列表所对应的DOM节点，将数据追加到列表中。但是，React框架和以前的写法完全不同，React是一个响应式的框架，强调不直接操作DOM，而是操作数据，对于数据的变化，React将自动感知并自动生成DOM。所以，如果使用React，我们将不再需要关注DOM操作，而是关注数据层的操作。

鉴于此，为了完成TodoList的功能，我们需要存储两个变量，一个变量存储输入框中的内容，另一个变量存储列表数据。因此，我们在TodoList组件中使用构造函数定义两个变量。

constructor(props) {
   super(props);
   this.state = {
        inputVaule:'',
        list:[]
     }
  }

这个构造函数接受一个固定的props参数。并在构造函数中调用super(props);方法。在React中，我们将数据定义在组件的状态对象this.state中，第一个变量inputValue存储输入框中的内容，第二个变量list用于存储列表数据。

因为我们用变量inputValue对应输入框中的内容，所以需要将inputValue和输入框进行绑定。根据React语法，在变量this.state.inputVaule的最外层添加一个大括号，并将值赋予输入框的value属性。

<input value={this.state.inputVaule} />

修改构造函数中的inputValue默认值，测试绑定是否有效。

constructor(props) {
   super(props);
   this.state = {
        inputValue:'hello!!!',
        list:[]
        }
  }

启动项目，此时输入框显示的内容为hello!!!，则表示我们绑定成功。

尝试在输入框中输入内容，输入框中的内容不变。显而易见，输入框中的内容由inputValue值所确定，此时inputValue被固定写死成了hello!!!。想要改变输入框中的内容，需要在输入框中绑定一个事件，当输入框中的内容改变时，将输入框内容更改为当前输入的内容。

<input value={this.state.inputVaule} onChange={this.handleInputChange} />

由于handleInputChange函数并不存在，我们在组件的最后定义这个函数。这么函数默认接收一个事件对象变量e。其中e.target代表输入框对应的DOM节点，e.target.value代表输入框中的内容。

handleInputChange(e) {
    console.log(e.target.value);
 }

打开浏览器控制台，可以看到输入框的当前内容。为了重新设置state里的inputVaule的值，将e.target.value赋值给变量this.state.inputValue。

handleInputChange(e) {
    this.state.inputValue=e.target.value;
 }

尝试在输入框输入内容，浏览器控制台抛出错误提示state变量未被定义，打印this变量，this变量的值等于undefined，说明this变量指向不对，因此，我们需要用bind(this)将this变量重新绑定到组件本身。

<input value={this.state.inputVaule} onChange={this.handleInputChange.bind(this)} />

再次打印this变量，这时this变量绑定到TodoList组件。再次尝试输入内容，还是报错。React规定，若需要改变state对象中的变量值，应该使用this.setState()方法，而不是使用this.state.inputValue=e.target.value;

handleInputChange(e) {
        this.setState({
            inputValue:e.target.value
        })
    }

再次尝试，输入框中的内容已经可以修改了。

整理一下代码，完整代码如下：

import React,{Component,Fragment} from 'react';

class TodoList extends Component{
    constructor(props) {
        super(props);
        this.state = {
            inputVaule:'',
            list:[]
        }
    }
    render() {
        return (
            <Fragment>
                <div><input value={this.state.inputVaule} onChange={this.handleInputChange.bind(this)} /><button>提交</button></div>
                <ul>
                    <li>学英语</li>
                    <li>Learning React</li>
                </ul>
            </Fragment>
        )
    }
    handleInputChange(e) {
        this.setState({inputVaule: e.target.value});
    }
}

export default TodoList;

# TodoList实现新增待办功能

前面，我们已经实现了输入框的功能，本节我们将实现TodoList新增列表项的功能：当我们在输入框中输入内容点击提交按钮后，输入框内容将加入到输入框下方的列表项中并清空输入框内容。为此，我们给提交按钮增加一个点击事件。

<button onClick={this.handleBtnClick.bind(this)} >提交</button>

通过handleBtnClick方法，我们展开列表中的已有内容并将刚输入的内容加入到state对象的list变量中，并清空输入框中的内容。

 handleBtnClick() {
        this.setState({
            list:[...this.state.list,this.state.inputValue],
            inputValue:''
        })
    }

在上一节中，列表数据在代码中写死了。为了能够动态添加列表数据，修改无序列表这一部分的代码，改为通过循环的方式动态展示列表数据。

<ul>
    {
       this.state.list.map(
         (item,index)=> {return <li>{item}</li>})
    }
</ul>

测试一下，依次输入1、2、3这些内容并点击提交，不出意外，列表成功新增数据。

查看Chrome浏览器控制台，出现警告信息。

根据警告信息，我们为列表的每一项添加唯一的key属性，并将其值设为列表项的序列号。

<ul>
    {
       this.state.list.map((item,index)=> {return <li key={index}>{item}</li>})
    }
</ul>

这样，我们就完成了TodoList组件的新增功能。

# TodoList实现删除待办功能

在完成新增功能以后，本节将实现列表项删除功能：通过删除功能，当我们点击列表项时，将调用删除方法，将对应数据从列表中移除。同样地，我们为每个列表项都增加一个点击事件，并将列表项的index值传递给处理函数，通过index值删除对应的列表项。

<ul>
    {
       this.state.list.map((item,index)=> {return <li key={index} onClick={this.handleItemDelete.bind(this,index)} >{item}</li>})
    }
</ul>

React不建议直接修改state状态对象中的值，因此，我们先将state状态对象中的list数据拷贝一份，移除index对应的列表项后再重新赋值给state状态对象中的list变量。

handleItemDelete(index) {
        const list = [...this.state.list];
        list.splice(index,1);
        this.setState({
            list:list
        });
 }

依次点击列表中的列表项，对应数据被成功删除。

# TodoList示例细节优化

通过上面的步骤，我们已经完成了TodoList组件的基本功能。本节将在现有功能的基础上做一些细节上的优化。

首先，我们为输入框增加一个红色边框样式。为此，在src目录下新建style.css文件，定义一个类选择器样式并在TodoList组件中引用它。

.input {
    border: 1px solid red;
}

在React中，为避免和类定义关键字class混淆，不推荐使用标签的class属性指定样式，而是使用className属性指定样式。

 <input className="input" value={this.state.inputValue} onChange={this.handleInputChange.bind(this)} />

接着，指定输入框和标签的id，为输入框添加标签标识。在React中，label标签不使用for属性，而是使用htmlFor，避免和循环关键字for产生歧义。

<div>
   <label htmlFor="insertArea" >输入内容</label>
   <input id="insertArea" className="input" value={this.state.inputValue} onChange={this.handleInputChange.bind(this)} />
   <button onClick={this.handleBtnClick.bind(this)} >提交</button>
</div>

然后，我们尝试在输入框中输入内容<h1>Li Mei</h1>并提交，结果在列表项中出现了内容<h1>Li Mei</h1>。

这显然不是我们想要的结果，为此我们为标签添加dangerouslySetInnerHTML属性保证html代码正常显示。

<ul>
   {
     this.state.list.map(
         (item,index)=> {
              return <li key={index} onClick={this.handleItemDelete.bind(this,index)} 
              dangerouslySetInnerHTML={{__html:item}} >
              </li>}
                  )
   }
</ul>

最后，再次尝试输入<h1>Li Mei</h1>，结果显示html代码已经可以正常显示了。

# TodoList示例完整代码

import React,{Component,Fragment} from 'react';
import './style.css'

class TodoList extends Component{
    constructor(props) {
        super(props);
        this.state = {
            inputValue:'',
            list:[]
        }
    }
    render() {
        return (
            <Fragment>
                <div>
                    <label htmlFor="insertArea" >输入内容</label>
                    <input id="insertArea" className="input" value={this.state.inputValue} onChange={this.handleInputChange.bind(this)} />
                <button onClick={this.handleBtnClick.bind(this)} >提交</button></div>
                <ul>
                    {
                        this.state.list.map(
                            (item,index)=> {
                                return <li key={index} onClick={this.handleItemDelete.bind(this,index)} 
                                dangerouslySetInnerHTML={{__html:item}} >
                                </li>}
                        )
                     }
                </ul>
            </Fragment>
        )
    }
    
    handleInputChange(e) {
        this.setState({inputValue: e.target.value});
    }

    handleBtnClick() {
        this.setState({
            list:[...this.state.list,this.state.inputValue],
            inputValue:''
        })
    }

    handleItemDelete(index) {
        const list = [...this.state.list];
        list.splice(index,1);
        this.setState({
            list:list
        });
    }
    
}

export default TodoList;

# React组件拆分与组件间传值

前面，我们已经完成了TodoList组件基本功能的编写，但是也要看到，现在我们只有一个组件TodoList。本节我们将拆分TodoList组件中的列表，将其中的列表项作为一个单独的组件进行编写。并通过组件拆分学习组件间传值和代码优化。

# 编写TodoItem子组件

为了拆分组件，我们在src目录下新建文件TodoItem.js，这个组件返回一个固定的文本内容item。我们将其导出以供外部引用。

import React ,{Component} from 'react';

class TodoItem extends Component {
    render() {
        return <div>item</div>;
    }
}

export default TodoItem;

# 引用子组件TodoItem

接下来，我们在TodoList组件中引用上面写好的TodoItem子组件，替换TodoList组件中原先编写的无序列表这一部分的代码，直接使用TodoItem组件。

<ul>
     {
       this.state.list.map(
            (item,index)=> {
             	return (
                    <TodoItem />
                      )
                    }
                 )
            }
</ul>

在输入框中输入内容并提交，可以发现无论我们输入什么内容，列表新增加的列表项内容都为item，这是因为先前TodoItem组件已经把返回内容写死了。

为了使列表添加的列表项内容能够按照输入的内容展示，我们需要让父组件向子组件传递数据。

# 父组件向子组件传值

在React中，父组件向子组件传递数据，是通过属性的形式向子组件传递的。在这里，当我们循环列表数据时，通过TodoItem子组件的属性将列表项数据传给子组件。我们使用content属性来传递列表项要展示的内容，当然也可以使用其它变量名来传递展示数据。

<TodoItem content={item} />

当子组件接收到父组件传递过来的数据时，使用this.props.属性名来动态获取数据内容。

import React ,{Component} from 'react';

class TodoItem extends Component {

    render() {
    return <div>{this.props.content}</div>;
    }

}

export default TodoItem;

尝试在输入框中输入内容并提交，增加的列表项内容已经可以动态展示了。

接下来，我们还需要实现动态删除列表项功能。因此，我们在TodoItem组件里绑定一个点击事件方法handleClick，并在构造函数中添加this.handleClick = this.handleClick.bind(this);代码将this变量指定为TodoItem组件，解决this变量值为undefined问题。

import React ,{Component} from 'react';

class TodoItem extends Component {
    constructor(props) {
        super(props);
        this.handleClick = this.handleClick.bind(this);
    }

    render() {
    	return <div onClick={this.handleClick}>{this.props.content}	     </div>;
    }

    handleClick() {
       
    }

}

export default TodoItem;

然后，我们开始实现TodoItem组件的动态删除列表项的功能。

# 子组件调用父组件方法

对于删除列表项来说，列表项数据都是来源于TodoList中的list变量中的数据。因此要删除列表项，实际上是删除列表中对应的数据。如果需要改变父组件TodoList中的内容，只能通过间接的方式更改。

handleItemDelete(index) {
     const list = [...this.state.list];
     list.splice(index,1);
     this.setState({
         list:list
     });
}

查看TodoList组件中删除列表项的方法可知，完成删除操作需要提供对应数据项的下标值。因此，我们传递一个index数据给组件TodoItem并通过deleteItem属性将父组件删除方法handleItemDelete传递给子组件，以便子组件调用。

<ul>
       {
          this.state.list.map(
              (item,index)=> {
                return (
                  <TodoItem content={item}  index={index} deleteItem={this.handleItemDelete} />
                      )
                   }
               )
          }
</ul>

和使用父组件传递给子组件变量方式类似，我们在子组件TodoItem中通过this.props.deleteItem调用父组件删除方法handleItemDelete。

handleClick() {
  this.props.deleteItem(this.props.index);
}

执行操作，this.props.deleteItem(this.props.index);代码报错。这是因为在子组件中执行this.props.deleteItem(this.props.index);相当于调用this.handleItemDelete，而子组件不存在这个方法。所以，我们需要在父组件传递方法的同时强制绑定this变量到父组件。

<ul>
      {
         this.state.list.map(
            (item,index)=> {
                return (
                        <TodoItem content={item}  index={index} deleteItem={this.handleItemDelete.bind(this)} />
                       )
                  }
              )
         }
</ul>

测试一下删除操作。这样，我们就完成了组件的拆分工作。

# TodoList代码优化

前面，我们已经完成了TodoList功能的编写。本节我们将对已有的功能进行优化，包括代码简化，性能提升等。

  1. TodoItem代码

    对于TodoItem组件，主要使用结构赋值对代码进行简化，并将函数bind(this)操作移到构造函数里，这样可以提高性能。

    import React ,{Component} from 'react';

class TodoItem extends Component {
    constructor(props) {
        super(props);
        this.handleClick = this.handleClick.bind(this);
    }

    render() {
        const {content} = this.props;
    return <div onClick={this.handleClick}>{content}</div>;
    }

    handleClick() {
        const {deleteItem,index} = this.props;
        deleteItem(index);
    }

}

export default TodoItem;

  2. TodoList代码

    和TodoItem代码优化类似，函数bind(this)操作移到构造函数里，以提高性能。根据React要求，使用推荐的方式重写方法内的逻辑即可。最重要的改动是将无序列表中的循环独立为一个函数，方便调用又不显得代码凌乱。

    import React,{Component,Fragment} from 'react';
import TodoItem from './TodoItem';
import './style.css';

class TodoList extends Component{
    constructor(props) {
        super(props);
        this.state = {
            inputValue:'',
            list:[]
        }
        this.handleInputChange = this.handleInputChange.bind(this);
        this.handleBtnClick = this.handleBtnClick.bind(this);
        this.handleItemDelete = this.handleItemDelete.bind(this);
    }

    render() {
        return (
            <Fragment>
                <div>
                    <label htmlFor="insertArea" >输入内容</label>
                    <input id="insertArea" className="input" value={this.state.inputValue} onChange={this.handleInputChange} />
                    <button onClick={this.handleBtnClick} >提交</button>
                </div>
                <ul>
                    {this.getTodoItem()}
                </ul>
            </Fragment>
        )
    }
    
    getTodoItem() {
       return this.state.list.map(
            (item,index)=> {
                return (
                    <TodoItem key={index} content={item}  index={index} deleteItem={this.handleItemDelete} />
                )
            }
        )
    }

    handleInputChange(e) {
        const value = e.target.value
        this.setState(() => ({
            inputValue: value
        }));
    }

    handleBtnClick() {
        this.setState((prevState) =>({
            list:[...prevState.list,prevState.inputValue],
            inputValue:''
        }));
    }

    handleItemDelete(index) {
        this.setState((prevState) => {
            const list = [...prevState.list];
            list.splice(index,1);
            return {list}
        });
    }
    
}

export default TodoList;
