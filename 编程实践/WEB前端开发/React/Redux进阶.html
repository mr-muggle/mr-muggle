# UI组件和容器组件

有的时候，当我们把一个组件的逻辑和渲染放在同一个组件时，维护起来会比较困难。因此，我们可以拆分这个组件为UI组件和容器组件。UI组件被称为傻瓜组件，容器组件被称为聪明组件。UI组件负责页面的渲染，容器组件负责逻辑的处理

# 拆分组件为UI组件和容器组件

接下来，我们将组件拆分成UI组件和容器组件。在src目录下新建一个UI组件文件TodoListUI.js，修改TodoList代码，引用UI组件并将TodoList组件中render函数中的代码以及Antd相关的代码移动到此组件中来。为了使程序正常运行，我们将this.state.xxx一律改为this.props.xxx。

  • TodoListUI组件

    import React,{Component} from 'react'
import {Input,Button,List} from 'antd';
class TodoListUI extends Component {

    render() {
        return (
            <div style={{marginTop:'10px',marginLeft:'10px'}}>
                <div>
                    <Input value={this.props.inputValue} placeholder='todo Info' style={{width:'300px',marginRight:'10px'}}
                    onChange ={this.props.handleInputChange} />
                    <Button type='primary' onClick={this.props.handleBtnClick} >提交</Button>
                </div>
                <List
                    style={{marginTop:'10px',width:'300px' }}
                    bordered
                    dataSource={this.props.list}
                    renderItem={(item,index) => (<List.Item onClick={(index) => {this.props.handleItemDelete(index)} }>{item}</List.Item> )} />
                </div>
        )
    }
}

export default TodoListUI;

  • TodoList组件

    import React,{Component} from 'react';
import 'antd/dist/antd.css';

import store from './store/index.js';
import {getInputChangeAction,getAddItemAction,getDeleteItemAction} from './store/actionCreators';
import TodoListUI from './TodoListUI';

class TodoList extends Component {

    constructor(props) {
        super(props);
        this.state = store.getState();
        this.handleInputChange = this.handleInputChange.bind(this);
        this.handleStoreChange = this.handleStoreChange.bind(this);
        this.handleBtnClick = this.handleBtnClick.bind(this);
        this.handleItemDelete = this.handleItemDelete.bind(this);
        store.subscribe(this.handleStoreChange);
    }

    render() {
        return (
            <TodoListUI 
            inputValue={this.state.inputValue} 
            handleInputChange={this.handleInputChange} 
            handleBtnClick={this.handleBtnClick} 
            handleItemDelete={this.handleItemDelete}
            list={this.state.list} />
        )
 }

# 改造组件为无状态组件

我们已经将组件拆分为UI组件和容器组件，现在我们将UI组件改造为无状态组件：

import React from 'react'
import {Input,Button,List} from 'antd';
const TodoListUI = (props) => {
    return (
        <div style={{marginTop:'10px',marginLeft:'10px'}}>
            <div>
                <Input value={props.inputValue} placeholder='todo Info' style={{width:'300px',marginRight:'10px'}}
                onChange ={props.handleInputChange} />
                <Button type='primary' onClick={props.handleBtnClick} >提交</Button>
            </div>
            <List
                style={{marginTop:'10px',width:'300px' }}
                bordered
                dataSource={props.list}
                renderItem={(item,index) => (<List.Item onClick={() => {props.handleItemDelete(index)} }>{item}</List.Item> )} />
            </div>
    )
}

export default TodoListUI;

如果一个组件只包含render函数，可以考虑将这个组件改造成无状态组件。

# Redux中发送异步请求获取数据

接下来，我们就模拟发送异步请求获取数据，在TodoList .js文件中，我们增加一个生命周期函数用于发送异步请求。这里我们发送请求获取数据，定义并调用在actionCreators.js文件中定义的initListAction方法创建action，将action发送给reducer。

componentDidMount() {
        axios.get('api/todolist').then((res) => {
        const data = res.data;
        const action = initListAction(data);
        store.dispatch(action);
        });
}

actionCreators.js文件中定义的initListAction方法如下，其中常量INIT_LIST_ACTION从文件在actionTypes.js里引入。

export const initListAction = (data) => ({
    type: INIT_LIST_ACTION,
    data
});

actionTypes.js中定义的常量INIT_LIST_ACTION如下：

export const INIT_LIST_ACTION = 'init_list_action';

接下来，我们在reducer中定义一个判断，当action的类型为INIT_LIST_ACTION，重新初始化数据列表。

if(action.type === INIT_LIST_ACTION) {
    const newState = JSON.parse(JSON.stringify(state));
    newState.list = action.data;
    return newState;
}

# redux-thunk中间件入门

前面，我们已经完成了todoList组件的基本功能，但是也应该看到项目其实存在着一些问题。当我们需要处理复杂逻辑和多个异步请求时，将多个异步请求直接写在componentDidMount()生命周期里会导致我们的代码过于臃肿，不好管理。

redux-thunk是Redux的中间件，通过这个中间件，我们可以在action中编写复杂的业务逻辑或异步请求。现在，我们安装Redux-thunk：

yarn add redux-thunk

接下来，修改src/store/index.js文件，引入Redux-thunk。

import {createStore,applyMiddleware,compose} from 'redux';
import reducer from './reducer';
import thunk from 'redux-thunk';

const composeEnhancers =
  window.__REDUX_DEVTOOLS_EXTENSION_COMPOSE__ ?   
    window.__REDUX_DEVTOOLS_EXTENSION_COMPOSE__({}) : compose;

const enhancer = composeEnhancers(
  applyMiddleware(thunk),
);

const store = createStore(reducer,enhancer);

export default store;

通过上面的代码，我们可以同时使用Redux-thunk，REDUX_DEVTOOLS_EXTENSION。

接下来，我们改造actionCreators.js代码，增加getTodoList函数，在这个函数中，我们发送异步请求获取数据，创建action，同时这个函数还可以接收dispatch函数变量并将action发送给store。

export const getTodoList = () => {
    return (dispatch) => {
        axios.get('api/todolist').then((res) => {
            const data = res.data;
            const action = initListAction(data);
            dispatch(action);
        });
    }
}

当我们修改好actionCreators.js代码后，修改TodoList.js里的代码，调用store.dispatch()方法。

componentDidMount() {
    const action = getInitList();
    store.dispatch(action);
}

# redux-saga中间件入门

在React开发中，redux-saga和redux-thunk的功能类似。在实际开发过程中也多使用这两个中间件，因此我们可以尝试使用redux-saga中间件。在使用之前，需要执行命令安装redux-saga。

yarn add redux-saga

接下来，我们需要修改src/store下的index.js文件，引入redux-saga。

import {createStore,applyMiddleware,compose} from 'redux';
import reducer from './reducer';
import createSagaMiddleware from 'redux-saga';
import todoSagas from './sagas';

const sagaMiddleware = createSagaMiddleware()

const composeEnhancers =
  window.__REDUX_DEVTOOLS_EXTENSION_COMPOSE__ ?   
    window.__REDUX_DEVTOOLS_EXTENSION_COMPOSE__({}) : compose;

const enhancer = composeEnhancers(applyMiddleware(sagaMiddleware));

const store = createStore(reducer,enhancer);
sagaMiddleware.run(todoSagas);
export default store;

可以看到，在这段代码中，我们引入了redux-saga中的函数，同时还引入当前目录下sagas.js文件的todoSagas。为此我们在src/store目录下创建sagas.js文件，并按照官方文档配置添加以下内容。

function* mySaga() {
}
  
export default mySaga;

然后，我们在actionCreators.js文件中添加getInitList供TodoList组件中的中componentDidMount周期函数调用。

export const getInitList = () => (
{
    type: GET_INIT_LIST
});

actionCreators.js使用了常量GET_INIT_LIST，所以，我们需要在actionTypes.js文件中定义常量GET_INIT_LIST。并在actionCreators.js引用，下面是常量的定义。

export const GET_INIT_LIST = 'get_init_list';

定义完这些之后，我们还需要在componentDidMount周期函数中调用action。

componentDidMount() {
    const action = getInitList();
    store.dispatch(action);
}

原本在componentDidMount()周期函数执行的异步请求也移到saga.js文件中，根据saga的规定，这个文件必须导出一个generator函数。

import {takeEvery,put} from  'redux-saga/effects';
import { GET_INIT_LIST } from './actionTypes';
import axios from 'axios';
import {initListAction} from './actionCreators';

function* getInitList() {
    try{
        const res = yield axios.get('api/todolist');
        const action = initListAction(res.data);
        yield put(action);
    }catch(e) {
        console.log("网络请求失败");
    }
     
}

function* mySaga() {
  yield takeEvery(GET_INIT_LIST,getInitList);
}
  
export default mySaga;

在这个文件的mySaga函数中调用了takeEvery函数，当它接收到GET_INIT_LIST类型的action后会调用getInitList方法，getInitList方法会发起请求，并调用put方法发送action给store，再由store转给reducer。由reducer返回数据。因此，reducer.js文件中增加如下代码。

if(action.type === INIT_LIST_ACTION) {
     const newState = JSON.parse(JSON.stringify(state));
     newState.list = action.data;
     return newState;
}

至此，我们成功使用redux-saga实现了项目功能。

# 使用react-redux完成TodoList组件

react-redux是一个第三方模块，通过这个模块，可以帮助我们更好地在react中使用redux。在使用之前，我们先安装一下：

yarn add react-redux

首先，我们重写src/store目录下的index.js文件，创建并导出store：

import { createStore } from 'redux';
import reducer from './reducer';

const store = createStore(reducer);

export default store;

其次，我们创建src/store下的reducer.js创建reducer。

import {CHANGE_INPUT_VALUE,ADD_TODO_ITEM,DELETE_TODO_ITEM} from './actionTypes';

const defaultState = {
    inputValue:'',
    list:[]
}

export default (state = defaultState,action) => {

    if(action.type === CHANGE_INPUT_VALUE) {
        const newState = JSON.parse(JSON.stringify(state));
        newState.inputValue = action.value;
        return newState;
    }
    
    if(action.type === ADD_TODO_ITEM) {
        const newState = JSON.parse(JSON.stringify(state));
        newState.list.push(newState.inputValue) ;
        newState.inputValue = '';
        return newState;
    }

    if(action.type === DELETE_TODO_ITEM) {
        const newState = JSON.parse(JSON.stringify(state));
        newState.list.splice(action.index,1) ;
        return newState;
    }

    return state;

}

在这里，我们判断action的type类型，根据类型返回不同的数据。并在src/store目录下创建actionTypes.js文件，定义action类型常量。

export const CHANGE_INPUT_VALUE = "change_input_value";
export const ADD_TODO_ITEM = "add_todo_item";
export const DELETE_TODO_ITEM = "delete_todo_item";

并在src/store目录下创建对应的actionCreators.js文件。

import {CHANGE_INPUT_VALUE,ADD_TODO_ITEM,DELETE_TODO_ITEM} from './actionTypes';

export const getInputChangeAction = (value) => ({
    type: CHANGE_INPUT_VALUE,
    value
});

export const getAddItemAction = () => ({
    type: ADD_TODO_ITEM
});

export const getDeleteItemAction = (index) => ({
    type: DELETE_TODO_ITEM,
    index
});

这些部分和前面大致相同。重点来看src目录下的index.js文件。

import React from 'react';
import ReactDOM from 'react-dom';
import TodoList from './TodoList';
import { Provider } from 'react-redux';
import store from './store';

const App =(
    <Provider store={store}>
        <TodoList />
    </Provider>
);

ReactDOM.render(App, document.getElementById('root'));

在这个文件中，我们从react-redux引入了Provider组件，它将TodoList组件作为Provider组件的子组件，并通过store属性连接了store，这样TodoList可以获得store的所有数据。这是react-redux提供的一个重要组件，再来看组件TodoList，即TodoList.js文件。

import React from 'react';
import { connect } from 'react-redux';
import {getInputChangeAction,getAddItemAction,getDeleteItemAction} from './store/actionCreators';

const TodoList = (props) => {

        const {inputValue,list,changeInputValue,handleClick,handleDelete} = props;
        return (
        <div>
            <div>
                <input value={inputValue} onChange={changeInputValue.bind(this)} />
                <button onClick={handleClick.bind(this)}>提交</button>
            </div>
            <ul>
                {
                    list.map((item,index) =>{
                        return (<li key={index} onClick={()=> {handleDelete(index)}} >{item}</li>)
                    })
                }
            </ul>
        </div>
        );
}

const mapStateToProps = (state) => {
    return {
        inputValue:state.inputValue,
        list:state.list
    }
}

const mapDispatchToProps = (dispatch) => {
    return {
        changeInputValue(e) {
            const action = getInputChangeAction(e.target.value);
            dispatch(action);
        },
        handleClick() {
            const action = getAddItemAction();
            dispatch(action);
        },
        handleDelete(index) {
            const action = getDeleteItemAction(index);
            dispatch(action);
        }
    }
}

export default connect(mapStateToProps,mapDispatchToProps)(TodoList);

在这个组件中，我们引入了connect用于负责组件连接store。

export default connect(mapStateToProps,mapDispatchToProps)(TodoList);

可以看到mapStateToProps函数定义了映射state中的字段和props字段的对应关系，mapDispatchToProps函数接收dispatch函数参数用于操作数据。

运行程序，可以看到我们已经可以正常增删改查了。到此，就已经学习完React、Redux的基本知识。
