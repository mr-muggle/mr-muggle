# 准备工作

为了实现高可用，需要做如下准备：

  1. 准备两台服务器

  2. 在两台服务器上安装nginx软件，安装方式见前面章节

  3. 在两台服务器上安装keepalive软件，这里为了方便，直接使用yum安装

    yum install keepalived -y

    安装完成之后，在/etc下生成keepalived目录，在该目录下存在一个配置文件keepalived.conf。

# 主服务器配置

  • 使用vi /etc/keepalived/keepalived.conf命令打开keepalived配置文件，并替换为如下配置：

    global_defs {
   notification_email {
     acassen@firewall.loc
     failover@firewall.loc
     sysadmin@firewall.loc
   }
   notification_email_from Alexandre.Cassen@firewall.loc
   smtp_server 192.168.31.171
   smtp_connect_timeout 30
   router_id LVS_DEVEL
}

vrrp_script chk_http_port {
  script "/usr/local/src/nginx_check.sh"
  interval 2 #（检测脚本执行的间隔）
  weight 2
}

vrrp_instance VI_1 {
    state MASTER
    interface enp0s3
    virtual_router_id 51 # 主、备机的 virtual_router_id 必须相同
    priority 100 # 主、备机取不同的优先级，主机值较大，备份机值较小
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 1111
    }
    virtual_ipaddress {
        192.168.31.222
    }
}

  • 在/usr/local/src目录下添加vrrp_script chk_http_port指定位置和名称的脚本文件。

    #!/bin/bash
A=`ps -C nginx –no-header |wc -l`
if [ $A -eq 0 ];then
 /usr/local/nginx/sbin/nginx
 sleep 2
 if [ `ps -C nginx --no-header |wc -l` -eq 0 ];then
 killall keepalived
 fi
fi

# 启动Nginx和Keepalived

在两台机器上分别启动nginx和keepalived：

  • 启动nginx，在nginx的sbin目录下，执行命令

    ./nginx

  • 启动keepalived

    systemctl start keepalived.service

# 测试高可用效果

  • 在浏览器输入192.168.31.222，此时服务器访问nginx主页。

  • 测试高可用，关掉主服务器nginx和keepalived，先关闭keepalived

    systemctl stop keepalived.service

    再关闭nginx

    ./nginx -s stop

此时，再次刷新页面，发现还能正常访问nginx，nginx主服务器被关闭，服务被切换到nginx从服务器。

# Keepalived配置文件详解

从上面的配置文件看，配置文件可以分为三大部分：全局配置、脚本配置，虚拟ip配置。

  • 全局配置

    global_defs {
   notification_email {
     acassen@firewall.loc
     failover@firewall.loc
     sysadmin@firewall.loc
   }
   notification_email_from Alexandre.Cassen@firewall.loc
   smtp_server 192.168.31.171
   smtp_connect_timeout 30
   router_id LVS_DEVEL
}

    其中，router_id为服务器的名字，可以在/etc/hosts中进行配置。

    127.0.0.1 LVS_DEVEL

  • 检测脚本配置

    vrrp_script chk_http_port {
  script "/usr/local/src/nginx_check.sh"
  interval 2 #（检测脚本执行的间隔）
  weight 2
}
    • script用于指定脚本位置
    • interval代表脚本执行的时间间隔，2代表每隔两秒执行一次
    • weight代表当前服务器的权重

  • 虚拟ip配置

    vrrp_instance VI_1 {
    state MASTER
    interface enp0s3
    virtual_router_id 51 # 主、备机的 virtual_router_id 必须相同
    priority 100 # 主、备机取不同的优先级，主机值较大，备份机值较小
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 1111
    }
    virtual_ipaddress {
        192.168.31.222
    }
}
    • state代表该服务器是主服务器还是备份服务器，主服务器则设置为MASTER，备份服务器则是BACKUP。
    • interface用于指定虚拟ip绑定到哪一个网卡中，interface后面是网卡的名称，可以使用ifconfig命令进行查看。
    • virtual_router_id代表主备机的ip主机，相当于唯一标识
    • priority代表主备机的优先级，主机值较大，备份机较小
    • advert_int表示每隔多长时间发送一个心跳，检测主机或者服务器是否还活着
    • authentication代表权限校验的方式
    • virtual_ipaddress配置虚拟ip，这里可以绑定多个虚拟ip
